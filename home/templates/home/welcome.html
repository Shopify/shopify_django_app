Welcome

This is an example Shopify application that shows what you can do with the Shopify API.

1. Make sure you've entered a valid API-Key and Secret

If you haven't registered your application yet, then you must do so to obtain an API-Key and Secret.

  • Shop owners can create applications for themeselves through their own admin (under Apps > ManageApps)
  • Shopify Partners create applications through their admin

After registering your application with Shopify, you need to edit the shopify_settings.py file, and insert your API-Key and Secret there. Then restart this application.

2. Make sure your callback URL is correct

The URL should be {{ callback_url }}, with the host portion of the URL pointing to whichever domain is currently hosting this application.
The callback URL of your app can be changed in the API screen of your Shopify admin interface.

3. Login to your shop

On login, the shop owner will be redirected to a Shopify URL containing the API-Key of the application. The shop will need to grant access to your application to give it access to its private admin data. After authenticating, the browser will be redirected to the shopify_app.views.finalize view, which creates a session using the API-Secret, and then redirects to the example dashboard.
The example view at home.views.index will get the shop's recent Orders and Articles through the Shopify REST API and list them with all of your Products. You can click on a link of one of your objects and will then be taken to the corresponding admin area of your shop.

Note: There are no examples for creating or editing objects, but you are also able to do that with the Shopify API!

4. Check out the source code of the home.views.index and its home/index.html template

You can analyze those files to see how the ShopifyAPI is used and get a glimpse of what's possible.

5. Read additional documentation

Become an expert.

  • API documentation (An overview of what you can do with the Shopify API.)
  • Wiki (needs its own API section) (Get more information and share your knowledge.)
  • Forum (needs its own API section) (Ask questions and see what others already wanted to know.)

6. Publish your application

This feature is not implemented yet, but it in the future it will allow you to register your application, so that will be shown in a public directory with a description, so that more users will take notice of it.
